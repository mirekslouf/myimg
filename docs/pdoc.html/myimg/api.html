myimg.api

Module: myimg.api

A simple interface to MyImg package.

>>> # Simple usage of myimg.api interface
>>> import myimg.api as mi
>>>
>>> # (1) Open image
>>> img = mi.MyImage('somefile.bmp')  # input image: somefile.bmp
>>>
>>> # (2) Modify the image 
>>> img.cut(60)                # cut off lower bar (60 pixels)             
>>> img.label('a')             # label to the upper-left corner
>>> img.scalebar('rwi,100um')  # scalebar to the lower-right corner
>>>
>>> # (3) Save the modified image 
>>> img.save_with_ext('_ls.png')  # output: somefile_ls.png

More examples are spread all over the documentation.

  1. MyImage key objects:
    • myimg.api.MyImage = single image = an image with additional methods
    • myimg.api.MyReport = multi-image = a rectangular grid of images
  2. MyImage additional applications:
    • myimg.apps = list of available applications
    • myimg.api.Apps = adding additional utils/apps to MyImage

 87class MyReport(myimg.objects.MyReport):
 88    '''
 89    Class providing MyReport objects.
 90    
 91    * MyReport object = a rectangular multi-image.
 92    * This class in api module (myimg.api.MyReport)
 93      is just inherited from objects module (myimg.objects.MyReport).
 94    
 95    >>> # Simple usage of MyReport object
 96    >>> import myimg.api as mi
 97    >>> # Define input images    
 98    >>> images = ['s1.png','s2.png']
 99    >>> # Combine the images into one multi-image = mreport
100    >>> mrep = mi.MyReport(images, itype='gray', grid=(1,2), padding=10)
101    >>> # Save the final multi-image               
102    >>> mrep.save('mreport.png')   
103    
104    Parameters
105    ----------
106    images : list of images (arrays or str or path-like or MyImage objects)
107        The list of images from which the MyReport will be created.
108        If {images} list consists of arrays,
109        we assume that these arrays are the direct input to
110        skimage.util.montage method.
111        If {images} list contains of strings or path-like objects,
112        we assume that these are filenames of images
113        that should be read as arrays.
114        If {images} lists contains MyImage objecs,
115        we use MyImage objects to create the final MyReport/montage.
116    itype : type of images/arrays ('gray' or 'rgb' or 'rgba')
117        The type of input/output images/arrays.
118        If itype='gray',
119        then the input/output are converted to grayscale.
120        If itype='rgb' or 'rgba'
121        then the input/output are treated as RGB or RGBA images/arrays.
122    grid : tuple of two integers (number-of-rows, number-of-cols)
123        This argument is an equivalent of
124        *grid_shape* argument in skimage.util.montage function.
125        It defines the number-of-rows and number-of-cols of the montage.
126        Note: If grid is None, it defaults to a suitable square grid.
127    padding : int; the default is 0
128        This argument is an equivalent of
129        *padding_width* argument in skimage.util.montage function.
130        It defines the distance between the images/arrays of the montage.
131    fill : str or int or tuple/list/array; the default is 'white'
132        This argument is a (slightly extended) equivalent of 
133        *fill* argument in skimage.util.montage function.
134        It defines the color between the images/arrays.
135        If fill='white' or fill='black',
136        the color among the images/arrays is white or black.
137        It can also be an integer value (for grayscale images)
138        or a three-value tuple/list/array (for RGB images);
139        in such a case, it defines the exact R,G,B color among the images.
140    crop : bool; the default is True
141        If crop=True, the outer padding is decreased to 1/2*padding.
142        This makes the montages nicer (like the outputs from ImageMagick).
143    rescale : float; the default is None
144        If *rescale* is not None, then the original size
145        of all input images/arrays is multiplied by *rescale*.
146        Example: If *rescale*=1/2, then the origina size
147        of all input images/arrays is halved (reduced by 50%).
148        
149    Returns
150    -------
151    MyReport object
152        Multi-image = tiled image composed of *images*.
153        MyReport object can be shown (MyReport.show) or saved (MyReport.save).
154    
155    Allowed image formats
156    ---------------------
157    * Only 'gray', 'rgb', and 'rgba' standard formats are supported.
158      If an image has some non-standard format,
159      it can be read and converted using a sister MyImage class
160      (methods MyImage.to_gray, MyImage.to_rgb, MyImage.to_rgba).
161    * The user does not have to differentiate 'rgb' and 'rgba' images.
162      It is enough to specify 'rgb' for color images
163      and if the images are 'rgba', the program can handle them.
164    '''
165    pass
