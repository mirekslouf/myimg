myimg.api

Module: myimg.api

A simple interface to MyImg package.

>>> # Simple usage of myimg.api interface
>>> import myimg.api as mi
>>>
>>> # (1) Open image
>>> img = mi.MyImage('somefile.bmp')  # input image: somefile.bmp
>>>
>>> # (2) Modify the image 
>>> img.cut(60)                # cut off lower bar (60 pixels)             
>>> img.label('a')             # label to the upper-left corner
>>> img.scalebar('rwi,100um')  # scalebar to the lower-right corner
>>>
>>> # (3) Save the modified image 
>>> img.save_with_ext('_ls.png')  # output: somefile_ls.png

More examples are spread all over the documentation.

  1. How to use myimg.objects:
    • myimg.api.MyImage = single image = an image with additional methods
    • myimg.api.MyReport = multi-image = a rectangular grid of images
  2. MyImage objects - frequent methods:
    • myimg.objects.MyImage.scalebar = a method to insert scalebar
    • myimg.objects.MyImage.caption = a method to add figure caption
    • myimg.objects.MyImage.label = a method to insert label in the corner
  3. MyImage objects - additional applications:
    • myimg.api.Apps = class for adding additional utils/apps to MyImage
    • myimg.api.Apps.FFT = an example of one utility = Fourier transform
  4. Additional utilities and applications:
    • myimg.plots = sub-package with auxiliary functions for plotting
    • myimg.utils = sub-package with code for specific/more complex methods
    • myimg.apps = sub-package with code for additional applications
    • myimg.apps.iLabels = app for immunolabelling (detection, classification, collocalization)

 98class MyReport(myimg.objects.MyReport):
 99    '''
100    Class providing MyReport objects.
101    
102    * MyReport object = a rectangular multi-image.
103    * This class in api module (myimg.api.MyReport)
104      is just inherited from objects module (myimg.objects.MyReport).
105    
106    >>> # Simple usage of MyReport object
107    >>> import myimg.api as mi
108    >>> # Define input images    
109    >>> images = ['s1.png','s2.png']
110    >>> # Combine the images into one multi-image = mreport
111    >>> mrep = mi.MyReport(images, itype='gray', grid=(1,2), padding=10)
112    >>> # Save the final multi-image               
113    >>> mrep.save('mreport.png')   
114    
115    Parameters
116    ----------
117    images : list of images (arrays or str or path-like or MyImage objects)
118        The list of images from which the MyReport will be created.
119        If {images} list consists of arrays,
120        we assume that these arrays are the direct input to
121        skimage.util.montage method.
122        If {images} list contains of strings or path-like objects,
123        we assume that these are filenames of images
124        that should be read as arrays.
125        If {images} lists contains MyImage objecs,
126        we use MyImage objects to create the final MyReport/montage.
127    itype : type of images/arrays ('gray' or 'rgb' or 'rgba')
128        The type of input/output images/arrays.
129        If itype='gray',
130        then the input/output are converted to grayscale.
131        If itype='rgb' or 'rgba'
132        then the input/output are treated as RGB or RGBA images/arrays.
133    grid : tuple of two integers (number-of-rows, number-of-cols)
134        This argument is an equivalent of
135        *grid_shape* argument in skimage.util.montage function.
136        It defines the number-of-rows and number-of-cols of the montage.
137        Note: If grid is None, it defaults to a suitable square grid.
138    padding : int; the default is 0
139        This argument is an equivalent of
140        *padding_width* argument in skimage.util.montage function.
141        It defines the distance between the images/arrays of the montage.
142    fill : str or int or tuple/list/array; the default is 'white'
143        This argument is a (slightly extended) equivalent of 
144        *fill* argument in skimage.util.montage function.
145        It defines the color between the images/arrays.
146        If fill='white' or fill='black',
147        the color among the images/arrays is white or black.
148        It can also be an integer value (for grayscale images)
149        or a three-value tuple/list/array (for RGB images);
150        in such a case, it defines the exact R,G,B color among the images.
151    crop : bool; the default is True
152        If crop=True, the outer padding is decreased to 1/2*padding.
153        This makes the montages nicer (like the outputs from ImageMagick).
154    rescale : float; the default is None
155        If *rescale* is not None, then the original size
156        of all input images/arrays is multiplied by *rescale*.
157        Example: If *rescale*=1/2, then the origina size
158        of all input images/arrays is halved (reduced by 50%).
159        
160    Returns
161    -------
162    MyReport object
163        Multi-image = tiled image composed of *images*.
164        MyReport object can be shown (MyReport.show) or saved (MyReport.save).
165    
166    Allowed image formats
167    ---------------------
168    * Only 'gray', 'rgb', and 'rgba' standard formats are supported.
169      If an image has some non-standard format,
170      it can be read and converted using a sister MyImage class
171      (methods MyImage.to_gray, MyImage.to_rgb, MyImage.to_rgba).
172    * The user does not have to differentiate 'rgb' and 'rgba' images.
173      It is enough to specify 'rgb' for color images
174      and if the images are 'rgba', the program can handle them.
175    '''
176    pass

190        '''
191        Class providing FFT objects.
192        
193        * FFT object = Fast Fourier Transform of an image/array.
194
195        >>> # Simple usage of FFT objects
196        >>> import myimg.api as mi        # standard import of myimg
197        >>> img = mi.MyImage('some.png')  # open an image using myimg.api
198        >>> fft = my.Apps.FFT(img)        # calculate FFT of the img object
199        >>> fft.show(cmap='magma')        # show the result
200        
201        Parameters
202        ----------
203        img : image (array or str or path-like or MyImage object)
204            The 2D object, from which we will calculate FFT
205            = 2D-DFFT = 2D discrete fast Fourier transform.
206
207        Returns
208        -------
209        FFT object.
210        
211        Technical details
212        -----------------
213        * FFT object, 3 basic attributes: FFT.fft (array of complex numbers),
214          FFT.intensity (array of intensities = magnitudes = real numbers)
215          and FFT.phase (array of phases = angles in range -pi:pi).
216        * FFT object is pre-processed in the sense that the intensity center
217          is shifted to the center of the array (using scipy.fftpack.fftshift).
218        * FFT object carries the information about calibration (pixel-size),
219          on condition it was created from MyImage object (the typical case).
220        '''
